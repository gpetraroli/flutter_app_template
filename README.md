# Tasks
A simple tasks application in Flutter integrating with Supabase with Riverpod state management.

## Features

- **Supabase**
  - Email/password sign up, login, and logout
  - Per-user task CRUD
  - Row Level Security (RLS) policies to ensure users can only access their own tasks

- **Riverpod**
  - Centralized authentication state management
  - Reactive task list and task detail providers
  - Built-in loading and error handling for async Supabase calls

## Requirements

You need to have an active Supabase instance.
1. Create a Supabase account if you don't have one already;
2. Create a new Supabase project;
3. Run the following SQL to create the database schema:
```sql
create table tasks
(
    created_at   timestamp with time zone default now() not null,
    updated_at   timestamp,
    title        text                                   not null,
    body         text,
    user_id      uuid                                   not null
        constraint todos_user_id_fkey
            references auth.users
            on update cascade on delete cascade,
    id           bigint generated by default as identity
        constraint todos_pkey
            primary key
        constraint todos_new_id_key
            unique,
    completed_at timestamp
);

alter table tasks
    owner to postgres;

create policy "Enable users to view their own data only" on tasks
    as permissive
    for select
    to authenticated
    using ((SELECT auth.uid() AS uid) = user_id);

create policy "Enable insert for users based on user_id" on tasks
    as permissive
    for insert
    with check ((SELECT auth.uid() AS uid) = user_id);

create policy "Enable users to update their own data only" on tasks
    as permissive
    for update
    to authenticated
    using ((SELECT auth.uid() AS uid) = user_id);

create policy "Enable delete for users based on user_id" on tasks
    as permissive
    for delete
    using ((SELECT auth.uid() AS uid) = user_id);

alter table tasks enable row level security;
```

## Run the project

1. Clone this repository;
2. Copy the .env file into .env.override and add your SUPABASE_URL and SUPABASE_ANON_KEY
3. Inside the project directory install dependencies by running:
```sh
flutter pub get
```
4. Run the project by:
```sh
flutter run
```
